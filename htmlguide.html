1 User guide

In this section we describe the application we developed from a user perspective. Section 1.1 discusses how the application can be set up and started. Section 1.2 discusses how to change some global options. Section 1.3 explains how the virtual camera can be controlled by the user. Section 1.4 describes how objects can be added to the environment. Section 1.5 describes the interaction with objects by the user. Finally, Section 1.8 discusses saving to and loading from files.

In this guide, we will occasionally refer to menus and menu items within them. These menus (File, Edit, etc.) can be found in the menu bar at the top of the application. Menus and menu items are referred to in bold text. A menu item will be referred to by first stating the name of the menu, followed by a right arrow, and then the name of the menu item. For instance, File → Load... refers to the Load... item in the File menu. Finally, many options can also be activated through a keyboard shortcut. These will be between brackets and emphasized.

This user guide is also available online at http://maikelsteneker.github.io/BraitenbergUserManual.

1.1 Preliminaries

Building the project from source can be done using maven. We recommend using an IDE, such as NetBeans, for this. This will retrieve some required libraries (specifically OpenGL support through JOGL) and build the entire application. Upon starting the application, an empty environment will be shown. An example of this can be found in Section 1.

The environment shown here consists of a floor and four walls surrounding it, but it does not contain any vehicles or other object. Adding these is discussed in Section 1.4.

1.2 Global options

Global options can be configured via Tools → Options (Ctrl+Shift+O). This will open the options window, as shown in Section 2.

The options in this window are global options that apply to everything within the environment. No confirmation is necessary; changing an option in the options window makes the result immediately visible in the main window.


Figure 2: The options window.

The options are as follows:

Vehicle randomness This refers to the behavior of the Braitenberg vehicles. Recall that vehicles move based on sensor input. Therefore, if the sensors connected to the wheels of the vehicle give no input, the vehicle will not move. When this option is activated, this behavior is overridden. When a vehicle gets no input from its sensors for a certain amount of time, it will start moving randomly. As soon as one of its sensors are triggered again, it will start moving according to its natural behavior.

Show bounding boxes for vehicles Collision detection is implemented by maintaining bounding boxes for vehicles. Such a bounding box is a rectangle in which the complete vehicle is included. This options shows these bounding boxes as transparent purple areas close to the ground. An example of this can be seen in Figure 3.


Figure 3: A vehicle with its bounding box (shown in purple).

Collision detection Collision detection prevents moving vehicles from passing through each other. Enabling this option will make any vehicle that hits another vehicle or one of the walls surrounding the environment stop. Vehicles that have collided will not be able to move on their own without some sort of interaction from the user (such as moving the vehicle, explained in Section 1.5.

Show walls This shows the walls surrounding the environment. Disabling this option might be useful to more easily see the vehicles when the camera is rotated.

Show bounding boxes for walls This shows bounding boxes for walls as green rectangles on the ground level. This may be useful to get an indication of when vehicles collide with a wall when the walls themselves are not shown. An example of a bounding box for a wall is shown in Figure 4.


Figure 4: The bounding boxes of a walls (shown in green).

Show temperature gradient This shows circular shapes around heat sources that indicate the temperature of the environment. An example of this is shown in Figure 5.


Figure 5: An example of a temperature gradient.

Selection mode This refers to the way selected objects are marked. By default, a transparent cylinder is used, but alternatively, an arrow can be shown above the selected object. Both options are shown side-by-side in Figure 6.

(a) A transparant cylinder  

(b) An arrow

Figure 6: The different selection modes.

1.3 Camera control

The environment can be viewed through a virtual camera. This camera can be controlled in a number of different ways.

1.3.1 Rotation

The camera is rotated by dragging the mouse (while keeping the mouse button pressed) over the screen. Horizontal movement corresponds to rotating horizontally. Vertical movement corresponds to rotating vertically. Vertical rotation is limited to prevent rotating underground or ending with a camera in which the ground is up. Horizontal rotation is not limited.

1.3.2 Zooming

The mouse wheel can be used to zoom in and zoom out on the scene. Scrolling up corresponds to zooming in; scrolling down corresponds to zooming out.

1.3.3 Moving

Moving around through the environment can be done using the WASD keys on the keyboard. These move the point the camera is aimed at over the ground. The W key moves in the direction the camera is facing. The S direction moves in the reverse direction. The A and D keys move to the left and right respectively.

1.3.4 Resetting the camera

At any time, the camera can be reset to its original position through View → Reset camera (Ctrl+0).

1.4 Adding objects

This section discusses how objects can be added to the environment. Objects include vehicles (which can be configured to include different sensors and different connections), light sources (which influence light sensors) and heat sources (which influence heat sensors).

1.4.1 Adding vehicles

Adding a vehicle to the environment is done via Edit → Add vehicle... (Ctrl+A). This opens the add vehicle window. Note that this window can only be opened if no other addition windows (for vehicles, light sources or heat sources) are currently opened. The add vehicle window is shown in Figure 7.


Figure 7: The add vehicle window.

In this window, a number of different properties of the vehicle can be configured.

Pre-set vehicle There are a number of pre-set profiles for vehicles. Such a profile contains the type of vehicle (e.g. two-wheel), both of its sensors and the assignment of sensors to motors. Examples of these include light-seeking (moving towards light), heat-fleeing (moving away from heat) and random (randomly accelerating or slowing down). Choosing a pre-set vehicle here sets all of these other options in the window, but the reverse also works. That is, a manually configured pre-set vehicle will be recognized as such.

Type of vehicle This is the type of the vehicle, affecting both how it looks and the way its sensors are connected.

Sensor 1/2 These are the sensors that the vehicle uses. Options include light (higher values when the sensor faces more light), temperature (higher values when the sensor is close to a heat source), random (pseudo-random variation in values) and dummy (no input at all).

Add random fluctuation This adds some random fluctuation to the sensors. This can be done to make sensors behave more like actual sensors instead of mathematically perfect sensors. Fluctuation can be activated per sensor independently for the sake of flexibility.

Left wheel The sensor that should be assigned to the left wheel. This means that the left wheel will rotate more if the sensor selected here gives higher values.

Right wheel Analogue to the left wheel.

Reverse sensor positions Swaps the sensors for the left and right wheels.

Angle The angle at which the vehicle is rotated initially.

Preview Whether or not to show a preview in the environment before the vehicle is added.

Add to environment and close This adds the vehicle to the environment and starts its behavior. The add vehicle window is closed at this point and the vehicle is selected (see also Section 1.5).

When the preview is enabled, the preview can be selected and moved around before permanently adding it to the environment (see also Section 1.5).

1.4.2 Adding light / heat sources

Adding a light source to the environment is done via Edit → Add light source... (Ctrl+L). This opens the add light source window. Note that this window can only be opened if no other addition windows (for vehicles, light sources or heat sources) are currently opened. The add light source window is shown in Figure 8.

In a similar way, heat sources can be added through Edit → Add heat source... (Ctrl+H). The resulting window looks exactly the same.


Figure 8: The add light / heat source window.

This window is very straight-forward. It simply shows the three coordinates of the light source (x,y,z, where z is the height axis) along with a checkbox to enable or disable a preview in the real environment. As with vehicles, this preview can also be selected and moved as described in Section 1.5.

1.5 Interacting with objects

The objects (vehicles, light sources and heat sources) in the environment can be selected, moved and deleted at any time. This process is explained in this section.

1.5.1 Selection

Objects are selected by clicking on them with the left mouse button. Both objects in the environment and previews can be selected. Selection may also be influenced by opening new windows or loading environments.

Selection is visually indicated in one of two ways: a transparant cylinder or an arrow. Both are shown in Figure 6. The cylinder, shown in Figure 6a, is the default option.

1.6 Movement

The selected object can be moved by holding the right mouse button and dragging the cursor over the screen. Note that the mouse cursor does not need to be on top of the object itself. Objects can only move at their current height level (typically just above the ground) and will move depending on the camera angle. Specifically, moving the mouse cursor up will move the object away from the camera, moving down will move the object towards the camera and moving sideways moves the object to the left or right from the camera perspective.

Note that collision detection has no influence on manual movement. That is, a user can move objects through each other and through the walls of the environment.

1.7 Deletion

The selected object can be deleted using Edit → Delete selection (Delete).

1.8 File management

Saving and loading environments can be done using File → Save... (Ctrl+S) and File → Load... (Ctrl+O) respectively. This will open a file selection window such as the one shown in Figure 9.


Figure 9: The file selection window (for loading an environment).

Files can be load or saved from any file in the file system. The default extension for these files is .env, but in principle any file name can be used. Environments include all objects, including vehicles and their full configuration, but no global options.
